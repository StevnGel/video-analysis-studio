# Git 提交信息规范

## 1. 提交信息格式

采用以下格式：

```
<类型>(<范围>): <简要描述>

[可选的正文]

[可选的脚注]
```

## 2. 类型标识

- **feat**: 新功能
- **fix**: 修复问题
- **docs**: 文档变更
- **style**: 代码格式（不影响功能）
- **refactor**: 重构（既不是新功能也不是修复）
- **perf**: 性能优化
- **test**: 测试相关
- **chore**: 构建/工具变更

## 3. 规则

- 标题控制在 50 字符以内
- 使用祈使语气（如 "Add" 而不是 "Added"）
- 末尾不加句号
- 标题与正文之间空一行
- 正文解释 **what** 和 **why**，不解释 **how**
- 可以在脚注中引用 issue 或 PR 编号

## 4. 示例

```
feat(video): 添加视频片段截取功能

实现用户选择起始和结束时间进行视频截取的功能。
解决用户需要手动使用其他工具进行简单剪辑的问题。

Closes #123
```

```
fix(playback): 修复播放进度条卡顿问题

在快速拖动进度条时，更新频率过高导致界面卡顿。
通过节流处理解决了此问题。
```
